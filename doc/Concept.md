# Concept: Порівняльний аналіз інструментів для локального Kubernetes

## Вступ
Стартап **AsciiArtify** готує PoC ML-сервісу (перетворення зображень у ASCII-арт).
Для локального Kubernetes розглядаються три інструменти: **Minikube**, **Kind**, **K3d**.
Мета — обрати оптимальний інструмент для швидкого PoC-розгортання.

## Характеристики

| Інструмент | Підтримувані ОС | Архітектура | Автоматизація / Зручність | Додаткові можливості |
|-----------|------------------|-------------|---------------------------|----------------------|
| **Minikube** | Win/macOS/Linux | VM або Docker | Просте встановлення, зручне CLI | Addons, Dashboard |
| **Kind** | Win/macOS/Linux | Docker | Легко робити багатовузлові кластери, YAML-конфіги | CI/CD інтеграція |
| **K3d** | Win/macOS/Linux | K3s у Docker | Дуже швидкий запуск, низькі ресурси | Інтеграція з Rancher |

## Переваги та недоліки

| Інструмент | Переваги | Недоліки |
|-----------|----------|----------|
| **Minikube** | Простий старт, Dashboard, гарна документація | Повільніший, потребує більше ресурсів |
| **Kind** | Легкий, швидкий, працює в Docker | Без GUI, більше CLI-налаштувань |
| **K3d** | Найшвидший старт, мінімальні ресурси, простота | Менше сумісності з деякими плагінами |
